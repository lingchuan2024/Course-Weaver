# CourseWeaver 前端 MVP 重设计说明

## 1. 现有问题

当前界面把 CourseWeaver 的内部数据结构直接展示给用户：原文块、coverage ledger、relations、note chunks 都是真实存在的中间产物，但普通学习者和评委并不知道这些对象分别解决什么问题。因此界面虽然能展示 pipeline，但不能证明“这个产品能用”。

主要问题如下：

- 课件不可见。左侧只有抽象 block 缩略图，用户无法确认笔记对应的真实 PDF 页面。
- “原文对照”概念不清。它实际应该服务于溯源，而不是作为主阅读对象。
- 笔记不可下载。生成结果如果不能导出，就不像一个完成的笔记工具。
- 知识关系不像树。关系卡片适合调试，不适合表达课程知识结构。
- “可信度检查”语义过重但动作不清。用户真正需要的是：哪里可能漏了、这段笔记来自哪页、哪些原文被合并或忽略。
- 顶部说明卡片占据首屏空间。MVP 应该让用户一打开就看到课件、笔记和知识树。

## 2. MVP 前端目标

新版前端定位为“课件到笔记的学习工作台”，而不是 pipeline 调试台。首屏必须回答三个问题：

1. 原课件是什么？
2. 生成出来的笔记能不能直接读和下载？
3. 这份笔记是否可追溯、是否漏掉重要内容？

因此前端保留以下核心内容：

- 真实 PDF 课件预览。
- 可阅读、可搜索、可下载的生成笔记。
- 课程大纲式知识树。
- 选中段落/知识点/原文块后的来源详情。
- 漏洞检查列表，优先展示 missing 和 uncertain。
- 简洁指标：页数、笔记章节数、知识点数、覆盖率。

以下内容不作为首屏主内容：

- 大段产品说明。
- 原始 block 账本全量堆叠。
- 面向开发者的 relation card 列表。
- 把 covered、merged、ignored 等英文状态直接暴露给普通用户。

## 3. 信息架构

首屏采用三栏：

### 左栏：课件原文

左栏展示真实 PDF 页面，默认跳到第一张有实质教学内容的页。上方提供页码切换，下方提供页面缩略列表。解析出的 block 只作为“解析层”折叠展示，用于点击后和右侧来源详情联动。

这样用户先看到真实课件，而不是抽象蓝块。

### 中栏：生成笔记

中栏是主阅读区，只展示成品笔记。顶部提供“下载 Markdown”按钮。笔记中的来源 chip 仍保留，但它的作用是跳转到对应原文页和右侧来源详情。

中栏不再混放知识关系和 coverage，以免打断阅读。

### 右栏：知识树与检查

右栏是工具区，使用 tabs：

- 知识树：按 note plan 展示课程大纲，每一节下面列出知识单元，以及从这一节发出的关系。
- 溯源：展示当前选中的原文块、对应知识点、coverage 状态和原文文本。
- 漏洞：优先列出 missing、uncertain，再列出 merged/ignored，作为人工复核清单。

知识关系不再用散乱卡片展示，而是挂在每个章节节点下面，形成“章节 -> 知识点 -> 关系”的树状结构。

## 4. 交互规则

- 点击 PDF 解析层 block、笔记来源 chip、漏洞列表条目，都会选中同一个 source block。
- 选中 source block 后，左栏跳到对应 PDF 页，右栏切到“溯源”并显示详情。
- 点击知识树中的关系目标，跳到对应笔记章节。
- 搜索框同时过滤笔记、知识树和检查列表。
- 下载按钮直接导出当前 `project.json` 中的讲义式笔记 Markdown，不依赖后端服务。

## 5. MVP 边界

MVP 不做复杂力导向图。课程知识关系更适合先用树表达，因为评委和学习者会优先关心学习顺序、前置依赖和对比关系。图谱可放到 7、8 月作为增强能力。

MVP 也不承诺 PDF 渲染与 block overlay 完全对齐。当前先用浏览器内置 PDF 预览保证课件可见，用解析层列表完成溯源。后续可引入 PDF.js，把 block bbox 直接覆盖到真实页面上。

## 6. 后续增强

- 用 DeepSeek/Kimi 直接抽取结构化知识树，而不是只改写 note chunks。
- 引入 PDF.js，实现真实课件上的可点击高亮框。
- 支持导出 PDF、Docx、Obsidian Markdown。
- 知识树增加折叠、拖拽排序和人工修正。
- 漏洞检查支持局部重生成。
